# 1.0 Release Audit

This document is the release inventory for `@sha3/code-standards` `1.0.0`.

## Exit Criteria

- No heuristic rule fails `code-standards verify` by default.
- `code-standards verify --strict` fails when warnings are present.
- `Biome` remains the only authority for visual formatting decisions.
- `biome.json` and `tsconfig.json` alignment checks are structural, not string-based.
- Template code stays simpler than the rules it demonstrates.
- Template README files read like maintainer documentation, not scaffold narration.

## Rule Inventory

| heuristic verify rules | warning | heuristic | verifier-specific | required positive + negative fixtures | must be documented | medium | keep only with evidence |

- `resources/ai/templates/*`

## Standards Admission Rule

No new rule may enter `resources/ai/rule-catalog.json` unless it ships with:

- a normative source in `standards/*`
- an implementation reference
- a positive fixture
- a negative fixture
- an explicit check that it does not compete with `Biome` or duplicate generic tooling
